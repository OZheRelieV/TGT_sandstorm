Основная логика решения лежит в директории src:
 - data_preparation.py <- подготовка данных на основе utils;
 - inference.py <- предсказание модели;
 - model_fitting.py <- обучение модели;
 - utils.py <- генерация фичей.

Основной файл main.py реализует обучение классификатора, если весов нет в директории models,
иначе - предсказание для тестового набора данных.

В директории data лежат исходные, а также обработанные (с новыми фичами) данные.

В логике решения обязательно наличие директорий data, т.к. обработанные данные сохраняются именно туда,
models, т.к. именно там должны храниться веса и логи обученного классификатора.

Вводить путь до тренировочного/тестового набора нужно с расширением, а для файла с предсказаниями - достаточно
ввести только название (УКАЗЫВАТЬ РАСШИРЕНИЕ ФАЙЛА С ПРЕДСКАЗАНИЯМИ НЕ НУЖНО!!!).
Пример ввода:
	Input path/file for train file >data/train.csv
	Input path/file for test file >data/test.csv
	Input path/file for prediction file >inference_test
Запускать код через консоль командой python main.py, находясь в соответствующей директории.

======================================================================================================================
Во время обучения классификатора оптимизировалась метрика roc-auc, которая равна 0.9983764948891442.
